# 開始手順

## 初日

1. `PROJECT_CONTEXT.md` に案件情報、承認者、対象範囲、リリース方針を記入します。
2. `docs/00_sources/README.md` に受領資料の版数と提供者を記録します。
3. 現行資料とコードを調査して `docs/01_as_is/` に残します。
4. 矛盾・未確認事項を `docs/open_questions.md` に登録します。
5. 仕様確定前の判断は `docs/assumptions.md` に明記します。

## 週次ヒアリング後

1. `docs/hearings/H-xxx.md` に議事録を残す。
2. 決定事項を `docs/decisions/DEC-xxx.md` に分離する。
3. 変更なら `docs/changes/CR-xxx.md` に影響分析を記録する。
4. 承認後に要求・要件・設計・追跡表を更新する。

## 実装開始条件

- 要件IDがあり、状態が `Approved` である。
- 正常系・異常系・境界値を含む受入条件がある。
- 既存機能、DB、API、運用への影響を確認済みである。

## 完了条件

- 要件・設計・実装・テストを追跡表でたどれる。
- テストと回帰確認が完了している。
- 設計書・運用資料・変更履歴が更新されている。
