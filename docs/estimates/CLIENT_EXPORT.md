# 顧客提示用見積の作成手順

## ファイルの役割

| ファイル | 用途 | 顧客への共有 |
|---|---|---|
| `internal_estimate_calculator.xlsx` | 内部工数、内部単価、難易度、提示用倍率を計算する内部用ブック | 禁止 |
| `client_estimate_share_template.xlsx` | 顧客へ提示する人日・人月・金額・前提だけを載せるブック | 可（責任者確認後） |

## 手順

1. 内部用ブックの `Parameters` で、1人日、1人月、顧客提示用時間単価を承認済みの値へ更新する。
2. `Difficulty` で8観点を評価し、難易度と提示用工数倍率を確認する。
3. `Internal Estimate` に要件IDごとの作業、内部工数、内部時間単価、根拠を入力する。
4. `Checks` がすべて `PASS` または責任者が確認済みの `CHECK` であることを確認する。
5. `Client Preview` のカテゴリ別「提示工数（人日）」「提示工数（人月）」「提示金額」を確認する。
6. `client_estimate_share_template.xlsx` をコピーし、案件名・見積番号・日付・カテゴリ別の値を**値として**貼り付ける。
7. スコープ、対象外、前提、有効期限、変更時の扱いを責任者が確認してから顧客へ共有する。

## 守ること

- 顧客向けブックへ、内部工数、内部時間単価、難易度の詳細、バッファ倍率を転記しない。
- バッファは、未確定仕様、品質確認、関係者調整、リリース上のリスクを含む見積上の余裕として、契約・見積条件で説明できる状態にする。
- 1.20〜2.00は内部工数に掛ける**最終倍率**であり、追加で120〜200%を上乗せする意味ではない。
- 単価・倍率・最終提示額は、エージェントが推測して決めない。責任者の承認を記録する。
